```python
def _get_recipe_or_404(session: Session, recipe_id: int) -> Recipe:
    recipe = session.get(Recipe, recipe_id)
    if not recipe:
        raise HTTPException(status_code=404, detail="Recipe not found")
    return recipe
```

HELPER FUNCTION - default group creation:
```python
DEFAULT_GROUP_NAME = "Ingredients"

def _ensure_default_group(session: Session, recipe_id: int) -> IngredientGroup:
    """Get existing default group or create one."""
    group = session.exec(
        select(IngredientGroup).where(IngredientGroup.recipe_id == recipe_id)
    ).first()
    if group:
        return group
    group = IngredientGroup(recipe_id=recipe_id, name=DEFAULT_GROUP_NAME)
    session.add(group)
    session.flush()
    return group
```

ENDPOINTS - Ingredient Groups:

1. POST /{recipe_id}/groups - Create group
   - response_model=IngredientGroupPublic, status=201
   - Verify recipe exists via _get_recipe_or_404
   - Create IngredientGroup with recipe_id and name
   - Return created group

2. GET /{recipe_id}/groups - List groups
   - response_model=list[IngredientGroupPublic]
   - Verify recipe exists
   - Select all groups where recipe_id matches, order by id
   - Return list

3. PATCH /{recipe_id}/groups/{group_id} - Update group
   - response_model=IngredientGroupPublic
   - Verify recipe exists
   - Get group, verify group.recipe_id == recipe_id (404 if mismatch)
   - Update name from payload
   - Return updated group

4. DELETE /{recipe_id}/groups/{group_id} - Delete group
   - status=204
   - Verify recipe exists
   - Get group, verify group.recipe_id == recipe_id
   - Delete all ingredients in group first (cascade in code, not DB)
   - Delete group
   - Return None

ENDPOINTS - Ingredients:

5. POST /{recipe_id}/ingredients - Create ingredient
   - response_model=IngredientPublic, status=201
   - Verify recipe exists
   - If group_id not provided or group doesn't exist, auto-create default group
   - Calculate display_order: count of existing ingredients in group
   - Create ingredient with provided data and calculated display_order
   - Return created ingredient

6. GET /{recipe_id}/ingredients - List all ingredients
   - response_model=list[IngredientPublic]
   - Verify recipe exists
   - Get all groups for recipe
   - Get all ingredients for those groups
   - Order by group_id, then display_order
   - Return list

7. PATCH /{recipe_id}/ingredients/{ingredient_id} - Update ingredient
   - response_model=IngredientPublic
   - Verify recipe exists
   - Get ingredient via session.get(Ingredient, ingredient_id)
   - Get ingredient's group, verify group.recipe_id == recipe_id (404 if mismatch)
   - Update only provided fields (model_dump(exclude_unset=True))
   - Return updated ingredient

8. DELETE /{recipe_id}/ingredients/{ingredient_id} - Delete ingredient
   - status=204
   - Verify recipe exists
   - Get ingredient, verify it belongs to a group for this recipe
   - Delete ingredient
   - Check if group is now empty, delete if so
   - Return None

ENDPOINT - Bulk Operations:

9. PUT /{recipe_id}/ingredients - Bulk replace all
   - response_model=IngredientsBulkResponse
   - Verify recipe exists
   - Get all existing groups and their ingredient IDs
   - Process each group in payload:
     - If group.id exists and in DB: update name, handle ingredients
     - If group.id is None: create new group (flush to get ID)
   - For each ingredient in each group:
     - If ingredient.id exists: update fields
     - If ingredient.id is None: create new
   - Delete groups not in payload
   - Delete ingredients not in payload (per-group)
   - Set display_order from position in ingredients array (0-indexed)
   - Commit
   - Return full state with IngredientGroupWithIngredients

CONTEXT.md decisions to honor:
- Default group name is "Ingredients"
- Empty groups are auto-deleted when all ingredients removed
- Deleting a group cascade-deletes all its ingredients
- Bulk operation is all-or-nothing (transaction)
- Gaps in display_order allowed
