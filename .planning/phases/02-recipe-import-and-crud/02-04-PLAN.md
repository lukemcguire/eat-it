  <action>
Add two PATCH endpoints to the recipes router:

1. PATCH /recipes/{recipe_id}/rating
   - Accept RecipeRatingUpdate schema
   - Update only the rating field
   - Return RecipePublic or 404
   - Status 200 on success

2. PATCH /recipes/{recipe_id}/notes
   - Accept RecipeNotesUpdate schema
   - Update only the notes field
   - Return RecipePublic or 404
   - Status 200 on success

Implementation pattern:
```python
@router.patch("/{recipe_id}/rating", response_model=RecipePublic)
def update_recipe_rating(
    *,
    session: Session = Depends(get_session),
    recipe_id: int,
    rating_update: RecipeRatingUpdate
):
    recipe = session.get(Recipe, recipe_id)
    if not recipe:
        raise HTTPException(status_code=404, detail="Recipe not found")
    recipe.rating = rating_update.rating
    session.add(recipe)
    session.commit()
    session.refresh(recipe)
    return recipe
```

Follow the same pattern for notes endpoint.

CRITICAL: These endpoints should ONLY update rating/notes respectively,
not other recipe fields.
