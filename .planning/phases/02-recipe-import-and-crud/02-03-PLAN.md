2. ParseError dataclass:
   - code: ParseErrorCode
   - message: str
   - suggested_action: str

3. ParseResult dataclass:
   - success: bool
   - data: Optional[dict] (parsed recipe fields)
   - error: Optional[ParseError]
   - duplicate_warning: Optional[dict] (set by endpoint, not parser)

4. RecipeParser class:
   - __init__(self, timeout: float = 30.0)
   - async parse_url(self, url: str) -> ParseResult
   - Use httpx.AsyncClient with follow_redirects=True
   - Use scrape_html(html, org_url=url, supported_only=False) for wild mode
   - Map scraper methods to dict: title, description, instructions,
     ingredients (list), prep_time, cook_time, yields, image, keywords

Error handling:
- httpx.HTTPError -> NETWORK_ERROR
- WebsiteNotImplementedError, NoSchemaFoundInWildMode -> MANUAL_ENTRY_REQUIRED

Import exceptions from recipe_scrapers:

  <action>
Add a GET /parse endpoint to the recipes router for URL parsing with duplicate
detection.

Endpoint: GET /parse?url=...

Flow per CONTEXT.md:
1. Check for existing recipe with source_url == url (duplicate detection at
   parse time)
2. Parse URL using RecipeParser
3. Return structured response:
   - On success: {success: true, data: {...parsed recipe...}}
   - On success with duplicate: add duplicate_warning: {existing_recipe: {...}}
   - On failure: {success: false, error: {code, message, suggested_action}}

Response model (Pydantic):
```python
class ParseResponse(BaseModel):
    success: bool
    data: Optional[dict] = None
    error: Optional[dict] = None  # {code, message, suggested_action}
    duplicate_warning: Optional[dict] = None  # {existing_recipe: RecipePublic}
```

Key implementation details:
- Query param: url (required, str)
- Import RecipeParser in endpoint
- Query existing recipes: session.exec(select(Recipe).where(Recipe.source_url == url)).first()
- Return existing recipe as RecipePublic if found

Add this to the existing recipes.py router file.
