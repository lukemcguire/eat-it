<task type="auto">
  <name>Task 1: Create recipe router with CRUD endpoints</name>
  <files>src/eat_it/routers/recipes.py</files>
  <action>
Create src/eat_it/routers/recipes.py with a complete CRUD router following
the SQLModel pattern from RESEARCH.md Pattern 1.

Required endpoints:

1. POST /recipes - Create recipe
   - Accept RecipeCreate schema
   - Return RecipePublic with 201 status
   - Use session.add() + session.commit() + session.refresh()

2. GET /recipes - List recipes with pagination
   - Query params: offset (default 0, ge 0), limit (default 100, ge 1, le 1000)
   - Optional query param: q (search in title/description)
   - Return list[RecipePublic]
   - Order by created_at desc

3. GET /recipes/{recipe_id} - Get single recipe
   - Return RecipePublic or 404

4. PATCH /recipes/{recipe_id} - Update recipe (partial)
   - Accept RecipeUpdate schema
   - Use model_dump(exclude_unset=True) for partial updates
   - Use sqlmodel_update() helper
   - Return RecipePublic or 404

5. DELETE /recipes/{recipe_id} - Delete recipe
   - Return 204 No Content on success
   - Return 404 if not found

Import the schemas from eat_it.schemas.recipe (created in 02-01).
Use Depends(get_session) for database sessions.

CRITICAL: Do NOT include rating/notes updating in PATCH - those have dedicated
endpoints in Plan 04 per CONTEXT.md "Separate annotation endpoints" decision.
  </action>
