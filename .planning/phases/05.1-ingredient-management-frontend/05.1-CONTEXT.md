# Phase 05.1: Ingredient Management Frontend - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Frontend interface for managing ingredient groups and individual ingredients
within recipes. Users can view, create, edit, delete, and reorder ingredients
and groups through a desktop-first editing experience.

</domain>

<decisions>
## Implementation Decisions

### Edit Mode Pattern

- **Entry:** Explicit "Edit" button in the header/ingredients section
- **Save:** Single "Save Changes" button triggers bulk PUT to
  `/recipes/{recipe_id}/ingredients`
- **Cancel:** Warn on unsaved changes when user cancels or navigates away
  ("Discard unsaved changes?" dialog)
- **Visual indicator:** Edit button swaps to Cancel + Save buttons; header
  shows "Editing Ingredients" or similar
- **Scope:** Desktop-first editing for MVP; mobile shows view-only mode

### Group Visibility

- **Single group:** Display as flat list with no group header
- **Multiple groups:** Show group names as section headers with ingredients
  nested beneath
- **Full group CRUD:** Create, rename, delete groups via inline actions
- **"Add Group" button:** At bottom of ingredients section
- **Delete confirmation:** "Delete group and its N ingredients?" before
  cascade-deleting
- **Group reorder:** Drag groups to reorder (desktop)
- **Move ingredients between groups:** Drag ingredient to different group
  section (desktop)
- **Empty groups:** Keep until explicitly deleted
  (⚠️ **Gap:** Backend Phase 04.1 auto-deletes empty groups; may need backend
  adjustment or frontend workaround)

### Reordering UX

- **Drag handle:** Grip icon (⋮⋮) on left of each row
- **Library:** @dnd-kit for drag-and-drop implementation
- **Scope:** Desktop only (mobile editing deferred)

### Mobile Layout (View Mode)

- **Ingredient display:** One row per ingredient, format "1 C. flour"
  (quantity+unit, then name)
- **Groups:** Section headers, same pattern as desktop but stacked vertically
- **No editing:** Edit button hidden on mobile; view-only for MVP

### Claude's Discretion

- Exact styling/spacing for ingredient rows and group headers
- Drop indicator animation during drag
- Accessibility (keyboard navigation for drag-drop)
- Error handling for save failures
- Loading states during save

</decisions>

<specifics>
## Specific Ideas

**Single group (flat list):**
```text
| ingredient | amount |
| flour      | 1 C.   |
| sugar      | 100 g  |
```

**Multiple groups (with headers):**
```text
Cake
| flour | 1 C. |

Frosting
| sugar | 100 g |
```

- Each group section has an "Add ingredient" button at the bottom
- "Add Group" button at the very bottom of the ingredients section
- Group names are tappable to rename inline
- Delete (X) button on group headers

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets

- `useRecipes.ts` hook pattern: TanStack Query mutations with
  `queryClient.invalidateQueries`
- `apiClient<T>()` function: Generic fetch wrapper with error handling
- `RecipeDetailScreen.tsx`: Current static ingredient display (table format)
- UI components: Card, Icon, Button, Input, Tag, StepNumber
- Layout components: Header, BottomNav, Sidebar, AppLayout
- TouchButton: 44px minimum tap target component

### Backend Integration Points

- `GET /recipes/{recipe_id}/groups`: List ingredient groups
- `GET /recipes/{recipe_id}/ingredients`: List ingredients
- `POST /recipes/{recipe_id}/groups`: Create group
- `PATCH /recipes/{recipe_id}/groups/{group_id}`: Update group
- `DELETE /recipes/{recipe_id}/groups/{group_id}`: Delete group (cascades)
- `POST /recipes/{recipe_id}/ingredients`: Create ingredient
- `PATCH /recipes/{recipe_id}/ingredients/{ingredient_id}`: Update ingredient
- `DELETE /recipes/{recipe_id}/ingredients/{ingredient_id}`: Delete ingredient
- `PUT /recipes/{recipe_id}/ingredients`: Bulk replace all groups+ingredients

### Schemas (from src/eat_it/schemas/ingredient.py)

- `IngredientPublic`: id, group_id, quantity, unit, name, preparation, raw,
  display_order, timestamps
- `IngredientGroupPublic`: id, recipe_id, name, timestamps
- `IngredientGroupWithIngredients`: group with nested ingredients
- `IngredientsBulkRequest`: bulk replacement payload
- `IngredientsBulkResponse`: bulk operation response

### Types to Create (frontend)

- `Ingredient` interface matching `IngredientPublic`
- `IngredientGroup` interface matching `IngredientGroupPublic`
- `IngredientGroupWithIngredients` for nested display
- `IngredientsBulkRequest` for save payload

### Patterns to Follow

- TanStack Query hooks in `src/hooks/useRecipes.ts` pattern
- API client via `src/lib/api.ts`
- Types in `src/types/recipe.ts` (create `src/types/ingredient.ts`)
- Component structure: `src/components/recipe-detail/` or new
  `src/components/ingredients/`

</code_context>
